    问题一:培训班有没有必要报?

           我自己是在大学期间报过培训班的,是在大三暑假期间的时候,但是我上了两个月就自己退出了没在去过,从而开始了我的自学编程之路.正因为我有过这段经历,所以嘛,我对这个问题是有发言权的.
           我记得在我刚刚上大学的时候,由于是头一次,对什么事情都比较好奇,每天晚上就会在群里看到,xxx公司去综合楼宣讲啦,然后大家就一窝蜂的去了,那个时候并没有意识到那是培训机构.现在想来,那时候是真的silly,那时候培训机构培训的东西琳琅满目,有Php的,Linux的,C++的,计算机网络的,Java的,C的.etc.具体什么公司我也忘记了,ps:怎么可能会忘记呢😉.那个时候大家报了培训班还觉得挺自豪,但是后来我才知道这就是”韭菜”.如何大家在大一,大二的时候接触了这些培训机构,千万不要去报,去听听还是可以,可以了解一下计算机的行业.
           我来讲讲培训班对于在校读书期间的大学生是怎么培训呢?就是在周六周日的时候,让我们去一个指定的地方去集中培训,但是其实时间都花在路上了,去了讲十几分钟,练一个小时,在休息一个小时,根本学不到什么东西,并且晚上回宿舍的时候在巨累.而且学的那点东西周一到周五又给忘光了.所以我在上了培训班两个月的时候就自动退出了.还浪费了上万大洋!
           但是,如果你是大三下班学期了,这时候我是非常建议你去报培训班的,但是不能盲目乱报,而是要报有实力有名气的培训班,这里的名气可不是X度一搜搜出来的排行,而是去接触哪些报过培训班,培训完并且对他的评价非常好,那么就可以报这种的.为什么我建议在大三下班学期报培训班呢,因为大四上半学期国庆那会就是秋招的时候,如果你在大四上班学期报培训班,那么是完全来不及的,并且据我的了解,面试的技术越来越深,以前会JSP就要你,现在会JSP你一定过不了简历筛选那一关.
           总结,大一大二完全不需要报培训班,那会有大把时间可以自己学编程,并且在暑假的时候去大厂实习,到毕业那个时候根本不愁没有工作,并且有非常大的几率是大厂.(啊,好像穿越到大一,认真学习啊.不过也不可能了,只能现在努力好好提高自己的技术深度吧🤣),那么新生如何自学编程呢,请看下一节😉.下一节也不知道多会能码出来哈哈.我并不会为为了发表文章而发表文章的!

我自学编程的一些经历以及总结(第二节)

我是如何自学编程的

至于这篇文章的初衷,我是想把我的经验分享给在校大学生,因为在我的那个时候并没有人讲出来这些经验,我的上n届学长or学姐们也没有把他们宝贵的经验分享出来.所以在大学初期我浪费了很多时间,如果有这些经验的话,对于那时候的我的帮助是无比巨大的,我相信我可以比现在走的更远.希望这些经验可以对哪些想在编程之路行走的朋友带来帮助.

### 一.背景介绍 在大学初期,游戏陪伴我走过了二分一的大学生活,可以说大一是LOL陪我走的,大二是DNF陪我走的.到了大三我才把精力放在编程上,非常感谢大三的我,要不我可能在大四下半学期找见一份工作.虽然不是BAT,TMD.但是如果你大一就开始学编程的话,进这些公司是非常容易的.切记:我只分享学习经验,并不分享人生经验,因为我也才即将步入社会😘

二.准备工作

学习不只是找到好的学习路径，掌握好的学习方法就可以了的，要成为优秀的学习者，是要求很多素质和习惯的。我长期以来都是一个懒散的人，三分钟热度，自律很差。我相信大部分人都这样。在意识到学习任务的艰巨后，我想的是我不能再放弃了，我觉得我年轻试错的阶段已经过了，经不起再失败了，于是我专门花了很长时间来调整自己的习惯和行为。这里我分享一些对我帮助最大的资源：

1.书

《习惯的力量》（The Power of Habit）。这本书让我意识到自己的坏习惯是需要科学的方法来改变的，一味地自责和焦虑是没用的。习惯的改变靠的不是主观意志力，而是 cue （不知道怎么翻译，刺激物？）和日常流程（routine）来维持的，所以我把学习场所从家换到了图书馆。而 cue 分为情绪，时间，场所，上一个行为等，这些都需要自己去定位查找。找到触发你的坏习惯的 cue，改变它。另外，每天都要有计划，计划会触发奖赏期待，让你更不容易回到坏习惯。最重要的是，关键好习惯的养成，会让你养成其它好习惯更容易。比如早起的习惯一旦养成，我又养成了早上学习最具挑战知识的习惯。请记住要有计划

《学习之道》讲师 Dr. Barbara Oakley 是奥克兰大学的工程学教授。她本来是学斯拉夫语言文学的，一直畏惧工科，数理学科成绩也很差，直到快 30 岁时，她才决定挑战自己去学工程学。听她讲学习过程是最有说服力的。这本书教会了我学习的方法,认识到了其实我在进行学习这项活动中产生的一系列问题并不是我自己独有的,而是普遍存在的.比如在学校的时候,你其实想学习,但是一想到从宿舍到图书馆这么麻烦就不想去了,所以又在浑浑噩噩了一天,那么番茄工作法是解决这种问题特别有效的方法.如果不想看这本书,那么我当然有更好的方法啦😉,Coursera 上有一门课叫 Learning How To Learn, 目前是世界上参与人数最多的一门 MOOC 课程。这门课免费。讲师是这本书的作者。另一位讲师 Dr. Terrence Sejnowski 是顶尖的神经科学专家。他会从脑科学和神经科学的实证角度告诉你为什么应该怎样学。下面讲下我对这门课程的一些理念的应用。
《刻意练习》,这本书像你解释了为什么天才是天才,他们真的是只凭借智力么?还有哪些天赋是小时候可以刻意培养的,但是一旦过了那个阶段即使在努力,也达不到那个境界,所以以后在选择行业的时候,不要头铁去选择那个行业.专家并不是与生俱来.

至于这三本书权威不权威,那么请移步豆瓣.

  • 那么我通过这写书,我提炼出来适合我的哪些方法呢
1.大脑的集中和发散模式

发散模式（diffuse mode）指的是注意力从工作对象处转移开来，放松下来，让灵感有机会出现。集中模式（focused mode）自然是指全部注意力放在工作对象上，高强度思考。学习时，我采用番茄工作法，每工作 25 分钟休息 5 分钟。然后每工作 2 到 3 个小时，出门散下步.这个习惯养成后，很多神奇的事情会发生。比如我在学习编程的过程中,遇见一种问题是,我和视频上的代码一摸一样啊,为啥人家的代码就能出来正确的结果,但是我怎么都出不来啊.结果因为这个问题我浪费了一天的时间.后来我遇见这种问题了,就去上个厕所,走一走,然后回来一看,已经恍然大悟了.

2,多运动

Dr. Terrence Sejnowski 讲解了为什么运动会促发大脑新的神经元的生长。高强度的脑力活动，你需要大脑保持最佳机能。但是遗憾的是我也没把这个习惯养成,因为我喜静不喜动,哈哈.太惭愧了,我都没有养成,为啥还要写出来呢.因为身体是革命的本钱,要不哪来工作996,生病ICU这句话.挣了钱在健健康康的花出去,可别花在医院.奥里给,我肯定会养成的!

3,刻意练习

这个习惯是上面那本刻意练习的书教会我的.比如我在学习冒泡排序或者二分查找,虽然原理很简单,一听就明白,但是一coding就不知道从哪里开始了,或者写出来好多错,得调试半天.请记住,学习很容易产生幻觉,只有自己能够独立理解并且实现出来才是自己东西.多在 codewars 和 LeetCode 上刷题（我主要是在 LeetCode 上做题），可以巩固刚学的知识。不用开会员,白嫖就行,你要是能把这些题背会,相信我进大厂指日可待.

4,间隔重复

一天花 6 个小时学一个主题，和用 6 天时间，一天花 1 小时来学习同一个主题相比，肯定是后者掌握的更好。当然重复不是简单的重读一遍。而是刻意回忆，测试自己还能不能想起来。

2.APP

好的睡眠对于学习来说肯定是特别重要,推荐 心潮APP 到了晚上打开这个APP,跟着它得指导进入深度睡眠,我经常是跟着做了三分之一就睡着了,就毫无感觉了哈哈.如果自己压力比较大并且还很焦虑和注意力不集中可以试试冥想,不要感觉很玄乎,很管用的.Headspace 是国外一个比较流行的冥想软件。它的语速很慢,听懂很容易.

三.自学之路开始!

  • 零基础启蒙

怎么开始启蒙呢?当然是做个网页啦,MDN web开发入门(https://developer.mozilla.org/zh-CN/docs/Learn/Getting_started_with_the_web),MDN 全称是 Mozilla Developer Network，你可以认为是 Web 方面的官方技术网站。这个教程会带着你建立一个网站。然后，你可以把你的网页发布在 GitHub 上。
做完这些你已经对Html,css,javascript这三种前端语言有所了解了.注意我是自学的后端,并没有对前端有深刻的了解,所以我对于前端只会一笔带过.

学习了前面文章中的入门级经验和知识后，你可能会有两种反应。

  • 一种反应可能是，你对编程有一点的兴趣了，甚至有一点点小骄傲，可能还会四处炫耀。我想说，请保持这种感觉，但是你也要清醒一下，上面的那些东西，还不算真正的入门，你只是入门了一条腿。

  • 另一种反应也可能是，你被吓着了，觉得太难了。感觉不是一般人能玩的，如果是这样的话，我想鼓励你一下–“无论你做什么事，你都会面对各式各样的困难，这对每个人来说都是一样的，而只有兴趣、热情和成就感才能让你不畏惧这些困难”。所以，你问问你自己，是否从中收获了成就感，如果没有的话，可能这条路并不适合你。如果有的话，哪怕一丁点儿，你也应该继续坚持下来。

    那么就让我们正式开始吧,我是先通过视频自学,然后通过书籍更加完善我的程序员的职业素养. 想要学好编程,那么信息搜集能力是必不可少的.你想学哪种语言就去某宝,某鱼去搜索关键字试试看,记住找最新的别都2020年了,你找一个2016年.比如你想学java那么就去搜索关键字,相信我,你会找见你想要的.要是你觉得哪些不靠谱,可以直接关注我的公众号[月便士],回复java关键字,就直接得到了.当你跟着这些视频做完以后,你的能力肯定得到了一个很大的提升,看完估计也得一年吧.切记不要边看边敲,看完一节在敲,虽然你可能刚刚开始得看好几遍,但是这种提升是十分巨大的.

    • 网络协议- 《TCP|IP详解 卷一》足够了,看完之后对于网路协议会让你有一个更深刻的了解,不想看书的可以阅读以下MDN的官方文档 HTTP 的文档(https://developer.mozilla.org/zh-CN/docs/Web/HTTP)。你需要知道 HTTP 协议的几个关键点：1）HTTP 头，2）HTTP 的请求方法，3）HTTP 的返回码。还有，HTTP 的 Cookie、缓存、会话，以及链接管理，等等，在 MDN 的这个文档中都有了。对于 HTTP 协议，你不需要知道所有的东西，你只需要了解这个协议的最关键的那些东西就好了。
    • 数据库设计-我还没有看过这方面的书,不过这里推荐慕课网的一个在线课程：数据库设计的那些事。每个小课程不过 5-6 分钟，全部不到 2 个小时，我相信你一定能跟下来。你需要搞清楚数据的那几个范式，还有 SQL 语句的一些用法。
    • 我看过一些经典书籍 《Effective Java》,还有一定看下 Google Guava 库(https://github.com/google/guava) ，《代码整洁之道》,《程序员的职业素养》

  • 有了书籍推荐怎么可能会少工具的推荐呢🤣

    • 编程的 IDE。传统一点的，你可以使用 Eclipse（教程https://www.runoob.com/eclipse/eclipse-tutorial.html）。当然，我推荐你使用 Intellij IDEA（教程https://legacy.gitbook.com/book/dancon/intellij-idea/details）。这两个工具都可以开发各种语言，但是主要用在 Java。如果你想玩得更时髦一些的话，使用 Visual Studio Code 也不错，这个工具潜力十足，用其开发 Python、JavaScript、Java、Go、C 和 C++ 都能得心应手（教程https://legacy.gitbook.com/book/jeasonstudio/vscode-cn-doc/details）。其实这篇文章就是用这个工具写的
    • 版本管理工具 用Git就好了啊.教程(https://github.com/JiapengLi/GitTutorial)

四.我们还需要培养自己一些别的能力

  • 英文能力,说来惭愧对于英文中听说读写,我只会读和写,老天语速一快我就懵逼了(我看英文视频没字幕8太行🤣).但是毋庸置疑英文能力是非常重要的.因为我们所有的计算机技术全部来自于西方国家，所以如果你要想成为一个高手的话，那么必须到信息的源头去。英文的世界真是有价值的信息的集散地。你可以在那里，到官网上直接阅读手册，到 StackOverflow 上问问题，到 YouTube 上看很多演讲和教学，到 GitHub 上参与社区，用 Google 查询相关的知识，到国际名校上参加公开课……,为了再次体现它的重要性,我想告诉你的是大四应聘所有国有银行都必须过四级,并且面试考验的技术那简直是太简单了.有了四级证书,你在看完这篇教程,已经进了.等哪天我的英文能力足够强了我在试着写一篇教程吧.
  • 问题的能力 不会提问就无法用那程序员必备的三大网站,那肯定是重要的呀.可以看这个网站（How To Ask Questions The Smart Way）[http://www.catb.org/~esr/faqs/smart-questions.html]
